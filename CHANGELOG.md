# Changelog

本项目遵循按日期记录变更的方式。由于当前尚未发布正式版本，版本号先使用 `0.x`。

## [0.2.0] - 2026-07-04

### Added

- 增加多 Lmi 集合支持：打开角色目录时自动加载同目录下所有 `*Lmi` 文件夹。
- 增加缺部位诊断：列出 Spt 使用但当前 Lmi 集合未注册的 Limb 名，方便查找缺失依赖。
- 增加「联动」开关：关闭后旋转/移动/缩放只影响当前部位，子部位保持世界位置。
- 增加「添加部位」功能：可向当前帧添加未使用部位，也可重复添加已使用部位。
- 增加「移除部位」功能：从当前帧移除选中的部位条目。
- 浏览器端到端测试扩展到 83 项，覆盖 FK 开关、footY 保持、添加/移除部位、多 Lmi 与缺部位诊断。

### Changed

- footY 自动重算改为增量模式：保存原存档 footY 校准，只应用编辑造成的变化量，避免 HFEX 角色导入游戏后悬空/下沉。
- zip 导出改为：1 个 Spt zip + 每个 Lmi 集合各 1 个 Lmi zip。
- PNG 位图键在附加 Lmi 集合中使用 `LmiFolder/N.png` 形式避免不同 Lmi 的图池编号冲突。

### Fixed

- 修复父部位缺失时武器/子部位只能旋转不能移动的问题。
- 修复单击选中部位也触发零移动重烘焙、产生脏数据的问题。
- 修复 ref 复用帧空白问题：无自身 `uz` 且 `refIndex>0` 时显示源帧画面，并禁止姿势编辑。
- 修复缩略图首次加载时可能为空的问题。
- 修复矩阵分解中 `xScale=0` 退化矩阵旋转丢失的问题。
- 修复全量重建时误把 HFEX 存档中的从属槽位强行同步到主槽位的问题。

### Verified

- `node test/all.js` 通过。
- `node test/browser/e2e.js` 83/83 通过。
- Taylor 数据验证：缺 Lucas 拳头 Lmi 时能诊断；父拳头缺失情况下武器移动 Δtx/Δty 正常。

- 贴图浏览、替换 PNG、新增变体、锚点/关节编辑器。
- 自写 zip 导出。
- 使用说明与进度记录。
